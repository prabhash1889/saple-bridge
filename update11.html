Update 11

Saple Bridge Improvement Plan

Synthesized from two parallel deep-dive analyses (frontend/UX and backend/architecture/security), cross-checked against the security audit, auditor report, and the Swarm v2 design docs. A leverage plan, not a rescue plan: close known risk first, then convert existing plumbing into visible features, then ship Swarm v2.

1

Security and data integrity

Unfixed findings from the 2026-07-14 security audit plus concrete bug candidates found in this pass. Small, well-understood, real risk reduction. Do first.

Audit F1

Workspace-root allowlist for project_path

M

Every command accepts an arbitrary project_path; containment only constrains the relative part. A renderer compromise gets arbitrary-root FS access plus two RCE-capable commands. Managed HashSet of roots, checked in the containment keystone. Highest-value hardening in the repo.

project.rs lib.rs
Data bug

Lock review decision writes

S

submit_review_decision does 4 read-modify-writes of tasks.json / sessions.json outside the cross-process lock while sidecars write the same files - a silent lost-update window. Reuse the locked RMW pattern from control_plane.rs.

review.rs
Audit F4

Reap verification process tree on timeout

S

Timeout kills only the shell; a spawned dev server survives. Extract the PTY Job Object into a shared module and attach after spawn.

review.rs pty.rs

Cap agent outcome sizes

S

parseAgentOutcome never bounds summary / changedFiles / decisions; a runaway agent writes multi-MB strings persisted and re-rendered each poll. Cap summary ~2KB, arrays ~200.

2

Performance and robustness debt

Fine today, degrades as projects, memory dirs, and swarm runs grow. Fix before later phases multiply the load.

In-memory index for memory notes

M

Every lookup, search, mention scan, and save walks and re-parses every .md file - O(N) per operation. A small id-to-path index invalidated by the existing watcher collapses this.

memory.rs watcher.rs

Atomic snapshot restore

S

Restore deletes the live memory dir then copies; a crash mid-copy loses it (backup is best-effort). Copy to a sibling temp dir and rename-swap.

memory.rs

Swarm state versioning / CAS

M

write_swarm_state is a blind full-state overwrite of renderer-held state. Add a version field checked in Rust as an interim step before the Phase 6 Rust scheduler makes this moot.

swarm.rs swarmStore.ts

Stop reading untracked files in git_status

S

Each untracked file is fully read just to count lines; repos with many untracked files make status slow. Count lazily in the diff view.

git.rs

Per-tab draft content in Files room

M

One global dirty flag; only the active tab holds content, so switching tabs with edits forces discard-or-lose. Hold per-tab drafts in a Map.

fileStore.ts editor/*

Memory autosave / dirty guard

S

Saving is manual and navigating notes discards edits silently; edits during the save await are clobbered. Dirty flag + confirm, or debounced autosave, and merge rather than overwrite after save.

memoryStore.ts MemoryEditor.tsx

Fix fileStore depth-8 prune ceiling

S

Tabs opened deeper than 8 levels can be silently dropped on layout restore. Union open files into the prune whitelist.

fileStore.ts

Deduplicate now_iso()

S

Identical ~35-line hand-rolled calendar math in two modules - drift risk. Extract one shared helper.

project.rs memory.rs

Extend the file watcher

M

Only 3 files are tracked; external edits to memory notes, review records, and mailboxes never trigger reloads. Generalizing this also becomes the substrate for the Phase 6 coordinator loop.

watcher.rs App.tsx

Exit criteria: memory ops no longer O(N) per action; no store silently loses edits on navigation; watcher covers memory, review, and mailbox files.

6

Swarm v2 (flagship)

Executes the already-written design in docs/swarm-cross-provider-orchestration.md and swarm-redesign.md - the largest capability jump available. Strict order 6.1 to 6.3, message bus in parallel, scheduler last.

6.1

Provider adapter layer (providers.rs)

L

Per-provider headless / resume / parse_result, replacing marker-scraping with parsed JSON envelopes (claude -p stream-json, codex exec --json, droid exec -o json) and pinned permission postures. Highest-leverage backend work in the repo.

pty.rs branches replaced; new agent-result event into swarmStore.ts
6.2

Resumable agent sessions

M

agent_send_turn runs the adapter's resume invocation as a new attempt in the same pane, turning rework and mailbox messages into actually-delivered turns instead of dead drops.

pty.rs swarmStore.ts
6.3

Git worktree isolation per agent

L

worktree create / list / remove / merge_preview so parallel builders stop editing one shared tree - the root cause of most swarm conflicts, and the pattern every comparable product uses. Review diffs scoped per worktree branch.

git.rs swarm launch pipeline, review scoping
6.4

MCP message bus in the sidecar

M

send_message / fetch_inbox tools in saple-mcp make the dead-drop mailboxes a live channel with zero per-provider integration - every agent CLI already talks to the sidecar over stdio.

../saple-mcp/src/tools/
6.5

Rust-side swarm scheduler

L

Move the DAG scan and completion state machine out of the renderer (today it dies with the window; P13 replay exists only to compensate). Renderer becomes a view. Supersedes the interim CAS from Phase 2.

new engine.rs, control_plane.rs records as substrate

Exit criteria: a swarm survives an app restart mid-run; rework turns are delivered as real resumed sessions; two builders run in parallel without touching each other's tree.

Quality, testing, and infrastructure

Runs partly in parallel: every phase lands tests with its code. These items are the dedicated debt-payoff pass.

Frontend

Component test harness

M

Add @testing-library/react + jsdom - the entire 14k+ line component tree is untested. First targets: CommandPalette mode machine, ReviewWorkspace decision flow, TaskDialog validation, KanbanBoard drag.

Frontend

Untested stores

M

reviewStore (biggest gap - decision/staging/diff-cache), providerStore readiness merge, themeStore, sshPresetStore, layout stores.

Frontend

Security-relevant libs

S

juneDispatcher.ts (external control surface), highlighter.ts, taskAgentPrompt.ts, useFocusTrap.ts.

Frontend

Pin documented invariants

M

Race invariants documented in prose (swarm commit-by-id, P13 replay, pendingTaskReviews consume) get regression tests; plus the shortcuts-parity test.

Rust

PTY lifecycle integration test

M

A gated #[ignore] test spawning cmd /c echo asserting kill/join/no-orphan - covers the highest-risk code in the crate (Job Object, duplicate-id race, write-after-close, reader retry).

Rust

fs_lock concurrency tests

S

Multi-thread contention on atomic_write; stale-sentinel stealing and timeout-proceed behavior of the cross-process lock.

Rust

Zero-test modules + gaps

S

browser.rs and diagnostics.rs have zero tests; snapshot create/restore, review timeout/kill path, and june_control auth / bad-JSON branches are unexercised.

Infra

CI gate

S

Required jobs: typecheck, vitest, cargo test before any tag-driven release.

Infra

Sidecar version stamping + hash freshness

S

Record which saple-mcp commit shipped in which Bridge version; replace the size+mtime staging check with a hash comparison (the self-flagged upgrade path).

Infra

Robust version bumping

S

The Cargo.toml bump regexes the first version line (fragile) and leaves Cargo.lock stale. Anchor it or use cargo set-version, refresh the lockfile.

Infra

Deduplicate MCP config healing

S

.mcp.json and mcp_config.json are written and healed in duplicate blocks; extract the shared merge helper.

Exit criteria: CI blocks releases on typecheck + both test suites; every module with a known race or destructive path has a pinning test.
