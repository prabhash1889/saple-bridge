RUNTIME / ONE ACTION

How work moves through the bridge

A task update, terminal keystroke, memory edit, or Git action uses the same five-stage operating loop.

User acts

A room captures intent: run, edit, move, save, review, or coordinate.

Store updates

Zustand applies responsive UI state and prepares a scoped command.

IPC crosses

invoke() sends data to Rust; Tauri events carry streams back.

Rust validates

Paths, commands, process state, and secrets are handled at the trust boundary.

Disk reflects

Atomic writes update project artifacts; the watcher reloads genuine external edits.

Atomic by default

Rust writes to a sibling temp file, then renames over the target. Per-path locks serialize writers inside the process.

External edits are first-class

A native watcher emits saple-file-changed for tasks, swarm state, and sessions. Fingerprints suppress the app’s own write echoes.

Multiple windows into one repo

Workspace instances can keep independent terminal layouts while sharing the same project-local .saple/* state.

MCP can share the contract

The bundled saple-mcp sidecar exposes local memory tools to external clients through generated MCP config.

TRUST / DEFENCE

Local-first does not mean trust everything

The repository is user-controlled input. Saple Bridge protects the machine and avoids corrupting the very files it is meant to coordinate.

1

Path containment

Rust resolves requested paths against the selected project and rejects traversal outside it before reads, writes, or file actions.

2

Secret isolation

Provider API keys live only in the OS keychain under the Saple Bridge account—not in Zustand, localStorage, JSON, or Markdown.

3

Process ownership

The PTY registry tracks native child processes. Closing the app kills children and joins output threads to prevent orphaned agents.

4

Restricted WebView

The Tauri CSP limits scripts, objects, frames, fonts, images, and connections to the packaged application and its IPC channel.

5

Safe persistence

Atomic rename prevents torn JSON. Write fingerprints distinguish genuine outside changes from notifications caused by the app itself.

6

Signed updates

Windows update artifacts are verified against the public updater key before install and restart. Store builds defer updates to the Store.

DELIVERY / DESKTOP

From source to installed app

Vite produces the renderer, Cargo compiles the host, and Tauri packages both with the MCP sidecar into native installers.

SOURCEReact + Rust

TypeScript UI and native modules live in one repository.

STAGEsaple-mcp

The sibling sidecar is built and copied with its target triple.

BUILDVite + Cargo

Static renderer assets and the native host compile independently.

BUNDLETauri 2

App shell, icons, capabilities, plugins, and external binary are packaged.

RELEASETagged CI

GitHub Actions builds signed updater artifacts and a draft release.

UPDATEVerify + restart

Published latest.json drives the in-app Windows updater.
