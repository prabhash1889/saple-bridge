SAPLE Bridge / Implementation roadmap

Connect first.
Then accelerate.

An eight-step plan ordered by leverage, not novelty. The work starts by joining Bridge to the control plane it already has.

Objective

Make agent control immediate and coherent without weakening local-first storage, explicit review or safety boundaries.

What this plan avoids
  • No second source of truth
  • No general workflow framework
  • No infinite-canvas rewrite
  • No unbounded autonomous spawning
Guardrails

Improve the seam, not the stack.

Every item should reuse existing stores, PTYs, rooms, files and MCP tools wherever possible.

REUSE

Build through the current stores, Swarm scheduler, Command Palette, Review room and MCP sidecar.

ONE TRUTH

Do not create another authority for agents, tasks, runs or artifacts.

LOCAL

Keep keys in the OS keychain and durable project state under .saple/.

BOUNDED

Preserve human review, concurrency caps and explicit approval for expansion.

TESTED

Add one runnable check for every non-trivial behavior introduced.

SHALLOW

Prefer a prompt or tool contract over another service or abstraction layer.

Priority runway

P0

Connect the existing control plane

Bridge sessions and MCP runs become one durable lifecycle.

Foundation+

Build

Route existing session transitions into MCP agent, run, event and artifact tools. Do not add an event bus.

  • Launch → register agent + create run
  • Status → update agent + append event
  • Completion → artifacts + finish run
  • Review → final outcome

Done when

  • Kanban and Swarm launch exactly one agent and run.
  • Restart reconciliation never duplicates the run.
  • Completion evidence appears inside Review.
  • An integration test covers launch through review.

P1

Automatic on-demand context briefs

Every launch points the agent at the smallest relevant context tool.

Foundation+

Build

  • Coordinator calls get_project_brief.
  • Task agents call get_task_context.
  • Role-specific work calls get_agent_brief.
  • Search memories only when a brief points to them.

Done when

  • Launches identify task, role and brief tool.
  • Prompts never dump the entire memory graph.
  • Prompt tests verify the correct contract.
  • Missing MCP surfaces one clear diagnostic.

Likely touchpoints

P2

Universal agent composer

One keyboard-first surface for launching, redirecting and recording.

Interaction+

Build

Extend the existing Command Palette with targets: selected agent, all agents, reviewer, new terminal, Kanban task, new swarm and project memory. Use OS dictation as voice v1.

Done when

  • One global shortcut opens it.
  • The destination is visible before sending.
  • Mail and launches reuse existing paths and limits.
  • Keyboard focus returns predictably.
  • No dependency is added.

P3

Structured agent outcomes

Completion becomes evidence, not merely a terminal marker.

Interaction+

Build

Record summary, changed files, test command/result, decisions and review need with the existing artifact model. Keep completion markers as fallback.

Done when

  • Swarm shows summaries without opening a terminal.
  • Review shows test evidence.
  • Bad outcome data cannot break completion.
  • Marker-only agents still work.

P4

Bounded review-and-rework loop

Rejected work gets one traceable route back to its builder.

Iteration+

Build

Add attempt, maxAttempts and lastReviewFeedback. Rejection posts to the builder mailbox and offers relaunch; exceeding the cap needs approval.

Done when

  • Feedback reaches the correct builder.
  • Retry includes prior context and feedback.
  • Attempts survive restart and cannot be bypassed.
  • The dependency scheduler still rejects graph cycles.

P5

Local preview and screenshot context

Close the visual feedback loop without building a browser platform.

Expansion+

Build

Add a loopback-only Preview panel with refresh, external open, capture and attachment to a task, swarm or mailbox.

  • Images stay in the project context area.
  • Attachment destination is explicit and accessible.

P6

Approved dynamic worker requests

Agents may request capacity; Bridge remains the executor.

Expansion+

Build

MCP records a request with role, provider, mission and dependencies. Bridge shows it for approval, then inserts approved work through the current scheduler.

Done when

  • Requesters cannot directly execute workers.
  • The approval shows model, mission, dependencies and pane count.
  • Rejected requests spawn nothing.
  • Caps and duplicate request IDs are enforced.

P7

SSH terminal presets

Make remote shells repeatable without claiming remote-workspace parity.

Remote+

Build

Save display name, SSH host alias, remote directory and provider command. Launch through the custom-command PTY and rely on the user’s SSH agent.

Done when

  • The final command is visible before launch.

Deliberate deferrals

Not now is a product decision.

These ideas add substantial surface area before the core lifecycle is fully connected.

Infinite canvas

Validate that navigation—not orchestration—is the dominant user problem first.

Bundled speech model

Measure OS dictation use before shipping Parakeet or Whisper.

GPT Realtime

Avoid recurring cost and a new audio trust boundary until voice proves useful.

True remote workspaces

Design a remote sidecar/protocol instead of scattering SSH branches throughout local code.

Arbitrary agent loops

Bounded rework and approved worker requests cover the valuable cases safely.

Definition of success

Direct from one composer, retrieve only needed context, inspect a structured outcome, run one bounded retry and review the evidence—while .saple/ remains the durable truth.
